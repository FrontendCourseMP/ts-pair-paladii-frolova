Библиотеки для валидации форм: Just-validate, Yup, Superstruct, Formik, Joi

Критерии: 
Производительность (скорость работы)
Функциональность (поддержка различных типов ввода и правил валидации)
мощность валидатора
Простота использования

Рейтинг: 
1. Yup (Богатейший набор валидаторов)
2. Joi (Поддержка асинхронной проверки, но 
много весит)  
3. Superstruct (быстрая валидация, подходит для
анализа ввода и больших объёмов данных)
4. Zod (простое линейное API, поддерживает преобразования, но но сожно описывать условную валидацию)
5. Just-validate (Очень простой API, легковесный, работает напрямую с DOM, не требует зависимостей. Но ограниченный функционал, слабая TypeScript поддержка, мало возможностей для кастомной валидации)

JustValidate. 

Точка входа в библиотеку: импортим 
import JustValidate from 'just-validate';
const validator = new JustValidate('#form'); 

Методы:
1) addField(field, rules, config) - определяет правила валидации для нового поля. 
2) addRequiredGroup - это делает новую группу полей, необходимой.
3) .revalidateField - метод ручного запуска валидации поля.
4) revalidate - метод запуска всей валидации формы вручную.
5) .refresh() - метод обновления всей формы — настройки полей, очистка сообщений/стилей/классов об ошибках.
6) removeField - метод удаления правил/событий/ошибок валидации из конкретного поля.
7) showerrors - метод ручного отображения ошибок.
8) .showSuccessLabels - метод ручного отображения меток успеха.
9) .destroy() - метод отключения валидации: удалить все слушатели событий/ошибки/сообщения/стили/классы.
10) .removeGroup(group: string) - метод удаления правил/событий/ошибок валидации из конкретной группы.
11) .setCurrentLocale(locale?: string) - метод изменения текущего местоположения.
12) .revalidateGroup - метод запуска валидации группы вручную. 
Методы, которые чаще могут понадобиться нам: addField(), onFail(), addRequiredGroup()


Классы использования:
1) errorFieldStyle - кастомные CSS-стили для недопустимых полей.
2) errorFieldCssClass - пользовательские CSS-классы для недопустимого поля.
3) errorLabelCssClass - пользовательский CSS-класс для некорректной метки.
4) successFieldCssClass, successFieldStyle, successLabelCssClass, uccessLabelStyle
Пользовательские и кастомные CSS-стили для валидной метки.
5) lockForm: false - если верно, заблокируйте форму во время валидации. 
6) errorsContainer - по умолчанию ошибки отображались рядом с недопустимым полем.
7) validateBeforeSubmitting: false
если это верно, это запустит валидацию и покажет ошибки для поля перед отправкой формы. 
8) submitFormAutomatically: true
если это верно, форма будет отправлена при успешном прохождении валидации.
9) testingMode: true
если это верно, он добавит атрибуты data-testid для облегчения тестирования.

Правила проверки: 
required — элемент обязательно необходим для успешной отправки формы; 
minLength — обязательная минимальная длина поля; 
email — проверка адреса электронной почты; 
number — проверка числа; 
minNumber — минимальная длина числа; 
maxNumber — максимальная длина числа.
